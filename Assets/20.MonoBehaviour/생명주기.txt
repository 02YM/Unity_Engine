# 생명주기
- Unity의 생명주기는 Unity 엔진이 게임 오브젝트와 컴포넌트를 실행할 때 호출되는 메서드들의 흐름을 의미합니다.
이를 이해하면 게임의 흐름 제어, 초기화, 업데이트 타이밍 등을 효과적으로 다룰 수 있습니다.

# 개요
- Unity는 스크립트 컴포넌트에서 특정 이름의 메서드를 자동으로 호출합니다.

# 주요 생명주기 메서드 순서
## 오브젝트 초기화 단계(한 번 호출)
- 이 단계는 씬이 로드되거나 게임 오브젝트가 생성될 때 호출됩니다.
	메서드								설명
Awke()									모든 컴포넌트가 활성화되기 전에 호출됨. 의존성 설정에 적합
OnEnable()								오브젝트가 활성화될 때마다 호출됨
Start()									첫 프레임이 시작되기 직전에 한 번 호출됨, 컴포넌트들이 모두 초기화된 이후 사용 권장

## 프레임 마다 호출되는 메서드들(반복 호출)
	메서드								설명
Update()								매 프레임 호출, 주로 입력 처리, UI 갱신, 상태 업데이트 등에 사용
LateUpdate()							Update() 이후 호출, 카메라 이동 등 업데이트 후에 처리할 작업에 사용
FixedUpdate()							일정 시간 간격(물리 프레임)으로 호출, Rigidbody 물리 처리에 사용

## 렌더링 직전 & 이후
	메서드								설명
OnPreRender()							카메라가 렌더링을 시작하기 전에 호출
OnRenderObject()						렌더링 중 호출
OnPostRender()							렌더링 완료 후 호출
OnGUI()									GUI 이벤트 프레임마다 호출됨(여러번 호출될 수 있음)

## 비활성화 및 종료 단계
	메서드								설명
OnDisable()								오브젝트가 비활성화될 때 호출
OnDestory()								오브젝트가 삭제되거나 씬이 종료될 때 호출\


# 전체 생명주기 순서 요약
